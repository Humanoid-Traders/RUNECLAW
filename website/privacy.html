Privacy Policy

RUNECLAW Telegram Bot — by Humanoid Traders
Last updated: May 31, 2026

RUNECLAW is an educational prototype built for the Bitget AI Base Camp Hackathon S1. It operates in simulation mode by default. No real funds are traded unless the operator explicitly enables live trading.

1. What Data We Collect

When you interact with the RUNECLAW Telegram bot, we may process the following data:

  • Telegram User ID and Chat ID — used solely for authentication and message routing. We do not collect your name, phone number, or profile photo.
  • Bot commands and messages — the text of commands you send to the bot (e.g., /scan, /analyze BTC/USDT) is processed in real time to generate responses. Command text is not stored permanently.
  • Trade decisions — when you confirm or reject a trade idea, that decision is logged locally on the bot server for audit purposes.

2. What Data We Do NOT Collect

  • We do not collect your real name, email address, or phone number.
  • We do not access your Telegram contacts, groups, or other chats.
  • We do not share any data with third parties for advertising or marketing.
  • We do not use cookies or web tracking of any kind.
  • We do not store exchange API credentials on any external server — all keys remain in the operator's local environment variables.

3. How Data Is Used

All data processing serves a single purpose: operating the trading bot for the authenticated user. Specifically:

  • Authentication — your Telegram Chat ID is compared against the operator-configured TELEGRAM_CHAT_ID to restrict access.
  • Audit logging — trade decisions, risk check results, and state transitions are logged to local JSONL files on the server for transparency and debugging.
  • LLM analysis — market data (not personal data) may be sent to an AI model provider (OpenAI, Qwen, or another configured provider) for trade analysis. No Telegram user data is included in LLM requests.

4. Data Storage and Retention

  • All logs are stored locally on the bot operator's server or container.
  • No data is transmitted to Humanoid Traders or any central server.
  • Log files can be deleted by the operator at any time.
  • Paper trading portfolio state is stored locally and can be reset with a single command.

5. Third-Party Services

RUNECLAW interacts with the following external services:

  • Telegram Bot API — for receiving commands and sending responses. Subject to Telegram's Privacy Policy.
  • Bitget API — for fetching public market data (prices, tickers, order books). No personal data is sent. Subject to Bitget's Privacy Policy.
  • LLM Provider (OpenAI, Qwen/DashScope, or other configured provider) — market data only is sent for analysis. No personal information is included. Subject to the respective provider's privacy policy.

6. Security

  • Access is restricted to the configured Telegram Chat ID. Unauthorized users are rejected.
  • Rate limiting (20 requests/minute) protects against abuse.
  • Exchange API keys (if configured) are loaded from environment variables only and never logged, displayed, or transmitted.
  • The bot runs as a non-root user in Docker containers.
  • All safety-critical state (circuit breaker, loss streak) is persisted with atomic writes.

7. Your Rights

As a user of the RUNECLAW bot, you can:

  • Stop using the bot at any time by simply not sending commands.
  • Request the operator to delete all logs containing your Chat ID.
  • Block the bot on Telegram to prevent any further interaction.

8. Children's Privacy

RUNECLAW is not intended for use by anyone under the age of 18. We do not knowingly collect data from minors.

9. Open Source

RUNECLAW is open source under the AGPL-3.0 license. You can inspect exactly what data the bot processes by reviewing the source code at github.com/Humanoid-Traders/RUNECLAW-AI-TRADER.

10. Changes to This Policy

This privacy policy may be updated as the project evolves. The "Last updated" date at the top of this page reflects the most recent revision. Continued use of the bot after changes constitutes acceptance.
