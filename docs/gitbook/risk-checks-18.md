# 18 Pre-Trade Risk Checks

Every trade idea generated by RUNECLAW must pass **all 18 independent checks** before it enters the pending queue for human confirmation. This page provides a quick-reference card for each check.

> **Fail-closed design:** If any single check fails, returns an error, or cannot be evaluated, the trade is **REJECTED**. There are no overrides, no force-execute flags, no backdoors.

---

## Check Summary

| # | Check | What it guards against | Default threshold |
|---|---|---|---|
| 1 | Circuit Breaker | Trading after catastrophic loss | Auto-trips on 5% daily loss or 10% drawdown |
| 2 | Position Size | Oversized single positions | Max 20% of equity per position |
| 3 | Daily Loss | Exceeding daily risk budget | 5% of equity |
| 4 | Maximum Drawdown | Exceeding total drawdown from peak | 10% from peak equity |
| 5 | Open Positions | Overexposure via too many positions | Max 5 concurrent |
| 6 | Risk/Reward Ratio | Low-quality setups | Min 1.2x reward-to-risk |
| 7 | Confidence Threshold | Low-conviction trades | Min 60% blended confidence |
| 8 | Correlation Blocking | Concentrated sector bets | Max 2 positions per correlation group |
| 9 | Loss Streak | Revenge trading after losses | Rejects at 3+ consecutive losses |
| 10 | Entry Price Sanity | Data errors producing invalid prices | Entry must be positive and non-zero |
| 11 | Stop-Loss Required | Trades without downside protection | SL must be set and differ from entry |
| 12 | Stale Data Guard | Acting on outdated analysis | Max 5 minutes since idea generation |
| 13 | Cooldown Timer | Trading too soon after a loss | 300 seconds after last losing trade |
| 14 | Portfolio Exposure | Total portfolio overexposure | Max 80% total exposure |
| 15 | Per-Symbol Exposure | Single-asset concentration | Max 20% per symbol |
| 16 | Volatility Guard | Trading in extreme volatility | ATR/price ratio below 6% |
| 17 | Liquidity Guard | Insufficient order book depth | Fail-open (skipped if no book data) |
| 18 | Macro Event Gate | Trading during macro events | Blocks during EVENT_LOCKDOWN and BLACKOUT |

---

## Detailed Breakdown

### 1. Circuit Breaker

```text
IF circuit_breaker_active THEN → REJECTED
```

The circuit breaker trips automatically when daily loss exceeds 5% or drawdown exceeds 10%. Once active, **all trades are blocked** until a human manually resets it via `/reset`. Auto-recovery is intentionally not supported.

---

### 2. Position Size

```text
IF proposed_notional > equity * 0.20 THEN → REJECTED
```

No single position can exceed 20% of current equity. On a $10,000 portfolio, maximum position is $2,000 notional.

---

### 3. Daily Loss

```text
IF daily_realized_loss + unrealized_loss > equity * MAX_DAILY_LOSS_PCT THEN → REJECTED + trip breaker
```

Includes both closed-trade losses and mark-to-market unrealized losses. A temporary adverse spike against open positions can trip this. Intentionally conservative.

---

### 4. Maximum Drawdown

```text
IF (peak_equity - current_equity) / peak_equity > MAX_DRAWDOWN_PCT THEN → REJECTED + trip breaker
```

Measured from the highest equity ever recorded. Resets only when equity makes a new high.

---

### 5. Open Positions

```text
IF open_position_count >= MAX_OPEN_POSITIONS THEN → REJECTED
```

Prevents overexposure. Default: 5 concurrent positions maximum.

---

### 6. Risk/Reward Ratio

```text
R:R = |take_profit - entry| / |entry - stop_loss|
IF R:R < 1.2 THEN → REJECTED
```

Ensures every trade has adequate reward potential relative to risk.

---

### 7. Confidence Threshold

```text
IF blended_confidence < 0.60 THEN → REJECTED
```

Double gate: ideas below 60% are filtered at the analyzer level and re-checked at the risk gate.

---

### 8. Correlation Blocking

```text
IF positions_in_same_group >= MAX_CORRELATION_PER_GROUP THEN → REJECTED
```

Correlation groups: BTC, ETH, ALT_L1, MEME, DEFI, L2, AI, CEX. Default: max 2 positions per group.

---

### 9. Loss Streak

```text
IF consecutive_losses >= 3 THEN → REJECTED
IF consecutive_losses >= 5 THEN → trip circuit breaker
```

Prevents revenge trading. At 5 losses, the circuit breaker trips for a full halt.

---

### 10. Entry Price Sanity

```text
IF entry_price <= 0 OR entry_price IS NULL THEN → REJECTED
```

Guards against data corruption or API errors producing invalid trade parameters.

---

### 11. Stop-Loss Required

```text
IF stop_loss IS NULL OR stop_loss == entry_price THEN → REJECTED
```

Every trade must have a valid stop-loss that differs from the entry price. No exceptions.

---

### 12. Stale Data Guard

```text
IF now() - idea.created_at > 300 seconds THEN → REJECTED
```

Prevents acting on analysis that is more than 5 minutes old. Market conditions change.

---

### 13. Cooldown Timer

```text
IF last_loss_time + COOLDOWN_AFTER_LOSS_SEC > now() THEN → REJECTED
```

After a losing trade, the system waits 300 seconds (5 minutes) before accepting new trades. Prevents emotional/reactive re-entry.

---

### 14. Portfolio Exposure

```text
IF (total_open_exposure + proposed_position) / equity > MAX_PORTFOLIO_EXPOSURE_PCT THEN → REJECTED
```

Total portfolio exposure (all open positions combined) cannot exceed 80% of equity.

---

### 15. Per-Symbol Exposure

```text
IF (existing_symbol_exposure + proposed_position) / equity > MAX_SYMBOL_EXPOSURE_PCT THEN → REJECTED
```

No single asset can represent more than 20% of portfolio equity.

---

### 16. Volatility Guard

```text
IF atr_14 / current_price > VOLATILITY_GUARD_ATR_PCT THEN → REJECTED
```

Rejects trades when ATR (as percentage of price) exceeds 6%. During extreme volatility, stop-losses are unreliable and slippage is high.

---

### 17. Liquidity Guard

```text
IF order_book_depth_within_band < proposed_position_size THEN → REJECTED
```

Checks cumulative order book depth to ensure the position can be filled without excessive slippage. **Fail-open:** if order book data is unavailable, this check is skipped (the other 17 checks still apply).

---

### 18. Macro Event Gate

```text
IF macro_state IN (EVENT_LOCKDOWN, BLACKOUT) THEN → REJECTED
```

Blocks all trades during major macroeconomic events:

| State | Window | Action |
|---|---|---|
| NORMAL | >24h from event | No restriction |
| PRE_EVENT_CAUTION | Within 24h | Warning logged |
| EVENT_LOCKDOWN | 30min before to 30min after | **REJECTED** |
| POST_EVENT_VOLATILITY | 30min to 4h after | Warning logged |
| BLACKOUT | Calendar error | **REJECTED** (fail-closed) |

Tracked events: FOMC, CPI, Core PCE, NFP, PPI, GDP, ISM PMI, Retail Sales, Jobless Claims, Fed Speech.

---

## Re-Check on Confirmation

When a human taps **Confirm** on a pending trade, all 18 checks run **again** against the current portfolio state. This catches:

- New positions opened between analysis and confirmation
- Daily loss limit reached by other closed trades
- Macro event entering lockdown window
- Stale data (>5 minutes since analysis)

If the re-check fails, the confirmation is rejected with the specific check that failed.

---

## Configuration Reference

All thresholds are configurable via environment variables:

```bash
MAX_POSITION_PCT=2.0              # Risk budget per trade (%)
MAX_DAILY_LOSS_PCT=5.0            # Daily loss cap (%)
MAX_DRAWDOWN_PCT=10.0             # Max drawdown from peak (%)
MAX_OPEN_POSITIONS=5              # Concurrent position limit
MIN_RISK_REWARD=1.2               # Minimum R:R ratio
MIN_CONFIDENCE=0.60               # Minimum blended confidence
MAX_CONSECUTIVE_LOSSES=5          # Losses before circuit breaker
COOLDOWN_AFTER_LOSS_SEC=300       # Cooldown period (seconds)
MAX_PORTFOLIO_EXPOSURE_PCT=80.0   # Total exposure cap (%)
MAX_SYMBOL_EXPOSURE_PCT=20.0      # Per-symbol exposure cap (%)
MAX_CORRELATION_PER_GROUP=2       # Max positions per group
VOLATILITY_GUARD_ATR_PCT=6.0      # ATR/price rejection threshold (%)
STALE_DATA_MAX_AGE_SEC=300        # Max age of trade idea (seconds)
REQUIRE_STOP_LOSS=true            # Mandatory stop-loss
```
